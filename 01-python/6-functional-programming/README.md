# Session 3.2 — Functional Programming for Reusability

> **Module 1 · Python Programming Fundamentals and Flow Control**

In Session 3.1 your code learned to repeat itself across a list. But what if that *same* loop or `if`/`elif` shows up in 12 different places? Today you learn to bottle a block of code, give it a name, and **reuse it forever**. Welcome to **functions**.

---

## What you'll learn

- Build a reusable block of code with **`def`**
- Feed data **into** a function using **parameters** and **arguments**
- Hand data back **out** of a function with **`return`**
- Make functions flexible with **default values** and **keyword arguments**
- Understand **local scope** — why variables inside a function don't leak out
- Combine functions with the loops + `if`/`else` from earlier sessions
- Avoid the three classic beginner errors: forgetting `()` when calling, using `print` instead of `return`, code after `return`
